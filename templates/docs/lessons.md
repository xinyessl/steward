# 本项目经验库 / 防回归（项目专属 · 开工前必读 · 收尾时回写）

> **本项目独有的"踩坑记忆"**——只属于这个项目的坑：特定样式/主题色、本项目的字段命名怪癖、这套技术栈/库的专属陷阱、某个模块的隐藏规则……随本项目走、提交进本项目 git。
> （**通用、跨项目都适用**的坑——如"分页是否生效""搜索条件交互是否统一"——记到 **Steward 全局经验库** `$STEWARD_LESSONS`，不要记在这里。）

## 规矩（dev agent 必须遵守）
1. **开工前**：本文件 + 全局 `$STEWARD_LESSONS` **两层都扫一遍**，避开已知坑。
2. **收尾时分流**：解决了非显然的坑 →
   - **只对本项目成立**（样式/主题色/本项目字段或模块怪癖）→ 记**这里**。
   - **换个项目也会犯**（通用交互/通用功能/通用栈陷阱）→ 记**全局** `$STEWARD_LESSONS`。
3. 显然的小改不用记；宁缺毋滥，只记真实发生、确认有效的解法。
