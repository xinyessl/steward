你是编排器，进入【自动驾驶 · 精简并行】。范围：`$ARGUMENTS`（空 = P0）。
一个开发 agent 干完一条（规约→实现→测试→真库冒烟），**跨模块并行**；质量靠开发自检 + 真库冒烟，**验收交给人**（没有评审/测试 agent）。**autopilot 不自动验收**——每条做到 `accept=wait` 就交给你。

## 一、按模块切分（并行单位）
按本项目的**功能模块**切分（见 spec 的 `module` 字段 / 「①本项目信息」）。互不相干的模块可并行（各只动自己目录 → 并行安全）。缺 spec → 开发 agent 现写。

## 二、跨模块并行 + 模块内串行
- 并行 `Agent(dev)`，并发 ≤3 个模块，各只动自己模块目录。
- 模块内串行：一条做完立刻冲下一条，避免两个 dev 改同一文件冲突。

## 三、每条 spec（一个 `Agent(dev)` 一气干完）
1. 缺 spec → 依「①本项目信息」现写，`status=ready`；歧义/业务规则拿不准 → `NEEDS-HUMAN`，跳过并记入待人清单。
2. 实现 + 写测试 + **连真库冒烟（若有库，禁止只 mock）** + 编译 + 跑测试。
3. 写持久层前**必对照「①本项目信息 → 真实库结构文件」**，按真实列名；列/表不存在 → `NEEDS-HUMAN`。
4. 全绿 → `docs/.state/<id>.json` 置 `accept=wait`（控制台 🔔 提醒你验收），跑 `node tools/board.mjs`。
5. 自检不过 / `NEEDS-HUMAN` → 对应节点置 `fail` 或记待人清单，**跳过继续别的，不堵整条线**。

## 四、停 + 总报告
范围内全部做到 `accept=wait`（待你验收）或记入待人清单 → **停**。
总报告：⏳ 待你验收清单（每条 spec + 真库冒烟结果）、⏸ 待决策清单（原因）、整体风险。

## 五、铁律
- 🚫 **禁止臆造库表列名**：写持久层前先核「①本项目信息 → 真实库结构文件」，不存在 → `NEEDS-HUMAN`。
- 真子代理 `Agent(dev)` 干活，编排器不写业务码；每步回写 `.state` + 跑 board，让控制台实时可见、🔔 提醒待人项。
- 不臆造业务规则（权限/留痕/金额/数据范围）→ `NEEDS-HUMAN`；复用现有表、按契约、界面照基线。
