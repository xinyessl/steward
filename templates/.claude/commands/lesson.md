---
把「$ARGUMENTS」（没写就用我们**刚在本会话解决的那个问题**）记成一条经验。

**第一步：判断该进哪一层（§4.7 两层经验库）：**
- **项目专属**——只对本项目成立（特定样式/主题色、本项目字段或模块怪癖、这套栈的专属陷阱）→ 写 **`docs/lessons.md`**（本项目，随项目提交）。
- **通用**——换个项目也会犯（分页/搜索/筛选交互是否生效统一、表单校验通用陷阱、某框架库的通用坑）→ 写 **`$STEWARD_LESSONS`**（`echo "$STEWARD_LESSONS"` 取路径，steward 全局、跨项目跨人共享）。
- 拿不准就先归**项目级**（宁可放小范围，别污染全局）。

**第二步：写入选定的那个文件：**
1. `cat` 目标文件看现有内容与最大编号。
2. 在「📚 教训明细」追加一条（编号顺延 L-NNN）：**现象**（一句话）/ **范围·模块** / **根因** / **解法** / **防复发** / **关联**（spec id / 提交 / 无）。
3. 有可复用检查点 → 往该文件「✅ 自检清单」加一条。
4. 用 Edit/Write 写回。⚠️ 全局库就是 `$STEWARD_LESSONS` 那个绝对路径文件，别在项目里另建。
5. 回我一句：`已记录到[项目级/全局] L-NNN：<现象>`。

只记**真实发生、确认有效**的解法；模糊/没验证的不记，宁缺毋滥。
