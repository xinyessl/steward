# Steward 操作指南（导入新项目 → 日常流程）

> 面向使用者。Steward = 多项目研发控制台:一处纳管所有项目、一句话派 AI 开发、所有项目以统一规约(spec)为据、任务按"进件批次"跟踪。
> 人只做四件事:**spec 评审 / 里程碑验收 / 上线放行 / 模糊点裁决**,其余文档驱动、自动流转。

---

## 〇、先懂这套模型(2 分钟)

**两条线,各司其职:**

| 线 | 是什么 | 在哪看 |
|---|---|---|
| **任务清单(进件批次)** | 我**要做什么 / 做到哪了**(日常主线) | 控制台左栏 |
| **系统说明书(spec)** | 系统**是什么样**(功能模块 + 规约/验收,给 AI/人理解全貌) | 左栏右上「系统说明书」按钮 |

**统一流程(任何输入都走这条):**
```
一坨需求/问题进来(飞书文档 / 终端贴文字 / 手动)
  → ① 先分析要不要动 spec(大特性→新建/改 spec;小改→改某条 AC;不涉结构→不动)
  → ② 拆成 N 条 task,归入一个【进件批次】(带 时间 + 来源)
  → ③ 你确认(spec 改动 + 任务拆分一起拍板)
  → 落地:合并 spec + 写任务清单 → 开发
```
- 无论大小**都出 task**;大的才额外动 spec。
- 速记/小事 = 手动往任务清单加一条。

## 一、导入一个新项目

### 1. 打开控制台
- **桌面客户端**:打开 Steward App(左上角能看到版本号)。
- 或**网页版**:项目目录跑 `bash tools/start.sh` → http://127.0.0.1:5178。

### 2. 分清两类项目
| 类型 | 场景 | 入口命令 | 事实源 |
|---|---|---|---|
| **存量项目** | 已有代码 | `/scan` 逆向抽 spec | 以**代码**为准 |
| **绿地项目** | 只有 PRD+原型、还没代码 | `/init` 搭骨架 → `/spec` 拆需求 | 以 **PRD+原型**为准 |

### 3. 「新增项目」(控制台右上)
1. 点 **「新增项目」**。
2. 填:**项目 id**(字母数字-_)、**名称**、**路径**(绝对路径)。
   - 存量 + 远程仓库:填 **Git 地址**,会自动 clone 到该路径。
   - 绿地:选**绿地**模式,选 **PRD 文件** + **原型目录**(会拷进 `docs/`)。
3. 确认后,Steward 自动给项目装好骨架:
   - `.claude/commands/`(/intake、/spec、/build… 全套命令)
   - `CLAUDE.md`(编排手册,自动填项目名/id)
   - 初始化 `docs/tasks.json`(任务清单)、`docs/specs/`、看板等。

### 4. 建立 spec 基线(在该项目的终端里跑)
- **存量** → `/scan`:按模块逆向把代码抽成 `docs/specs/<ID>.md` 草稿(便宜模型、增量) → 你评审 `draft → ready`。
- **绿地** → `/init`(定栈+搭最小工程骨架) → `/spec`:把 PRD 正向拆成 spec 树 → 你评审。

> 评审 = 在「系统说明书」里看这些 spec 对不对,点头即作为事实源。

### 5. 团队协作(关键:让同事也有)
- **spec(`docs/specs/*.md`)、任务清单(`docs/tasks.json`)、命令(`.claude/commands/`)都入库** → 同事 `git clone` 就有。
- 你这边新建/改完后:在各项目 `git add . && commit && push`。
- 同事拿到项目:
  - `git clone` → 控制台「新增项目」指向该目录(会自动补齐命令/方法论);
  - 或对已有项目点项目操作区的 **🔄 同步方法论**(从内置模板覆盖刷新命令 + CLAUDE.md,保留项目自己的①信息)。
- 同事的 **claude CLI 要够新**(旧版不加载项目命令 → /intake 等会 "Unknown command";`npm i -g @anthropic-ai/claude-code` 更新)。

## 二、日常流程(项目跑起来之后)

### 1. 进件:把一堆需求/bug 变成任务
- **飞书文档**:先在项目操作区 **☁️** 配好该项目的飞书机器人凭据 → 终端 `/intake <飞书链接>`。
- **直接贴**:终端 `/intake` 后贴一段文字;或左栏「**手动记一条任务**」加单条。
- claude 会按统一流程:拉内容 → **先析是否调 spec** → **拆成任务批次**(每条关联 specRef) → **给你确认**(spec 改动 + 任务拆分) → 你拍板后落地。
- **你只在这一步拍板**:确认方向对、spec 改动对,就放行。

### 2. 看进度
- **左栏任务清单**:按进件批次分组(来源/时间/完成度),状态筛选(全部/待办/进行中/已完成),手动加、双击改、点状态切换。
- **「进度」按钮**:① 任务清单完成情况(主)② spec 文档验收情况(参考)。
- **「系统说明书」**:看 spec 全貌,顶部**搜索框**按 功能/模块/编号 快速定位;点单条看详情(可展开/收起全部章节)。

### 3. 开发
- `/build <spec-id>`:派一个开发 agent **一气做完**——补 spec(若缺)→ 实现 → 写测试 + 连真库冒烟 → 编译+跑测试,全绿才交付。
- `/fix <缺陷>`:定位 → 改 spec/测试 → 改码 → 回归 → 出报告。
- 做完 claude 会把对应 **task 回写为已完成**;涉及 spec 的同步更新 spec。

### 4. 验收(你的关卡)
- 全绿后控制台 **🔔** 提醒你验收。
- `/accept <spec-id>`:看验收报告 + 真库冒烟关键返回 + **spec diff**(涉及 spec 时)→ **通过**(spec 置 accepted)或**打回**(开发自动接手修)。

### 5. CLI 对话即编排(随口说也不丢)
- 在项目终端直接跟 claude 聊:它会**对照整体项目**把对话里的东西落地——
  - 确认了**行为/结构变更** → 直接更 spec;
  - 冒出**新需求/bug** → 直接加进任务清单(来源 cli);
  - 拿不准会先问一句。

## 三、人只守这四关
1. **spec 评审**(draft → ready)
2. **里程碑验收**(看演示 + 报告 + 拍业务正确性)
3. **上线放行**
4. **模糊点裁决**(响应 `NEEDS-HUMAN`)

## 四、控制台速查

**顶栏**:项目切换 · 改名 ✏️ · 飞书配置 ☁️ · 同步方法论 🔄 · 删除 🗑️ · 新增项目 · 进度 · 经验库 · 刷新
**左栏(任务清单)**:状态筛选 · 手动加任务 · 批次折叠 · 「系统说明书」
**终端工具条**:专注(折叠左栏) · 命令(贴命令) · 新窗口 · 历史对话 · 复制

**常用命令(在项目终端)**:`/intake`(进件,日常入口) · `/scan`(存量抽 spec) · `/init`(绿地搭骨架) · `/spec`(拆需求成 spec) · `/build`(开发) · `/fix`(修缺陷) · `/accept`(验收) · `/tasks`(整理任务) · `/lesson`(沉淀经验)

## 五、小贴士
- **复制(桌面端)**:claude 鼠标模式下,**按住 Option 拖拽**选中文字 → 自动进剪贴板 → `Cmd+V`;或选中后点「复制」。整屏直接点「复制」。
- **通知**:首次到「系统设置 → 通知」允许 Steward;需你确认时(后台/最小化)会弹系统通知。
- **数据在哪**:项目注册表/凭据在 `~/.steward/`;各项目内容(spec/任务/代码)在各自目录、随 git 走。客户端只读不带数据。
- **版本号**:左上角「文档驱动研发 · 多项目 · vX.Y.Z」。
