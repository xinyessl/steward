# Steward 全局经验库 / 防回归（通用·跨项目·跨人共享）

> **目的：解决过一次的通用问题，别在任何项目再犯。** 这是 **Steward 工具自身沉淀的"踩坑记忆"**——随 steward 仓库走、提交进 git，**跨所有项目、跨所有使用者共享**。
> **只记通用的坑**：换个项目也会犯的那种——分页/搜索/筛选交互是否生效统一、表单校验通用陷阱、某框架/库的通用坑、跨项目的接口/分页约定……
> **项目专属的坑**（特定样式/主题色、本项目字段或模块怪癖）→ 记到**那个项目的 `docs/lessons.md`**，不要记在这里。
> 文件位置：steward 仓库根的 `lessons.md`（环境变量 `$STEWARD_LESSONS` 指向它）。`cat "$STEWARD_LESSONS"` 读、Edit 写回。

## 规矩（dev agent 必须遵守）
1. **开工前**：先扫一遍下面的「✅ 自检清单」+ 与本次改动相关的「📚 教训明细」，避开已知坑。
2. **收尾时**：本次若踩了一个**非显然**的坑（查了一会儿才搞定、或很可能再犯的那种），**追加一条教训**（格式见文末模板），并把其中可复用的检查点**提炼进「✅ 自检清单」**。
3. **判断要不要记**：显然的小改不用记；值得记的是「下次别人/AI 很可能再踩同一个坑」。
4. 验收/复验时，人会顺带看一眼新增的教训对不对——错的就删，好的留。

## 🎨 通用 UI 规范（列表 / 筛选 / 表格）
> 适用于所有前端项目的列表页 / 后台管理界面。新页面默认照此实现，已有页面逐步对齐。（来源：病案平台 pams 反馈，沉淀为通用规范。）

1. **菜单分级分组**：侧边/导航菜单按业务域**分组分级**（父分组 + 子菜单，可展开/折叠），不要一长串平铺罗列。
2. **菜单稳定排序**：菜单项按固定 `sort` 排序，顺序**稳定且符合业务优先级**；不要依赖插入顺序或随机顺序。
3. **表格内部滚动，禁止整页滚动**：列表页表格区域**自身内部滚动** + 表头 sticky；页面外壳固定视口高度，**不出现 body 全局滚动条**。骨架：外壳 `height:100vh;box-sizing:border-box;overflow:hidden;display:flex;flex-direction:column`；工具条/分页 `flex:0 0 auto`；表格外层 `flex:1;min-height:0;overflow:auto`；`thead th{position:sticky;top:0}`。
4. **搜索/筛选条件可收缩**：表头筛选区**超过一行**时做成可展开/收起——默认只显示第一行 + "展开/收起更多"，避免 2+ 行筛选挤占垂直空间。
5. **内容卡片紧凑**：主内容卡片 **padding 收紧**，减少大块留白（避免默认大 padding 把表格挤到很小）。
6. **造数验证分页**：每个模块交付时**造一批足够触发多页的样本**，端到端验证分页（翻页、每页条数、总数）正确，而不是只有几条看不出问题。
7. **统一"返回门户"入口**：多子应用 + 门户架构下，**每个子应用都要有统一的"返回门户"入口**（prod 跳同源门户 `/`、dev 跳门户开发端口）；不要有的有有的没有。
8. **列表搜索条件持久化**：列表页的搜索/筛选条件 + 分页（页码、每页数）要**记录**——从列表进详情再返回时，**原筛选与所在页要保留**，不被重置。可用路由 query 同步 / `keep-alive` 缓存 / 状态 store。
9. **起止日期用单个范围框**：开始/结束这类成对条件做成**一个范围控件**（"起 ~ 止"），**不要拆成两个独立筛选格**——否则栅格换行会把一对拆到两行。label 用合并名（申请起/申请止 → "申请日期"）。
10. **栅格筛选列数统一**：同一系统所有列表页筛选栅格**列数一致**（如一行 4 个），列宽对齐；折叠时也保持满行；"展开/收起"只在确实超一行时出现，不足一行不显示折叠按钮。
11. **选择类筛选即选即查**：下拉/单选/多选/日期(范围)/可搜索下拉等"选择型"筛选项，**选中即触发查询**（`@change` → 重置到第 1 页 + 重新请求），不用再点"查询"。只有**自由文本输入框**才靠回车/搜索按钮触发。所有列表页、所有子系统统一。
12. **时间统一格式 `yyyy-MM-dd HH:mm`**：所有**展示**处（列表/详情/抽屉/卡片）的日期时间一律格式化成 `yyyy-MM-dd HH:mm`（24 小时制），不要直接显示后端原始值（ISO/带毫秒时区/时间戳）。用一个**共享格式化函数**统一处理（兼容 ISO/时间戳/Date），空值显示占位 `—`；纯日期字段可只到 `yyyy-MM-dd`。

## 📚 教训明细
> dev agent 把新教训追加在这里。

### L-001 搜索条件少的表格页，筛选按钮另起一行浪费纵向空间、视觉割裂
- 范围/模块：通用 · 前端列表/表格页布局
- 根因：默认把筛选条件区和「查询/重置」按钮分两行排，字段少时显得空、割裂。
- 解法：搜索条件少（≤2~3 个）时，把按钮**直接紧跟在搜索框后面、同一行**，不另起一行。
- 防复发：列表页布局先数搜索条件数量——少就单行（按钮跟随）；多（或带高级筛选）才换多行/可折叠。
- 关联：UI 规范，无 spec。
